Design to web

What are responsive websites?

Responsive websites are websites that are easily readable from any device. On a responsive site the content including images and tables, etc will simply adjust its size to the screen you are using to read the site. So no matter if you are reading the site on your mobile, your tablet or if you are looking at it on a desktop, you don't have to awkwardly scroll around and read tiny writing on a small mobile screen or massively huge writing on your desktop.

It is quite important because you want to be able to find information fast and read content comfortably on any device, no matter where you are at the moment.

What is mobile first design?

Mobile first design is exactly as it sounds: you start building your content for the smallest screen first and then scale up to bigger screens, so eventually you cover the layout for all screen sizes.

The idea behind this is to design the hardest layout for the tiniest screen first, so you have all essential features first. From there you can take the idea and add new features on to make a site look good on larger screens too. This should help you keep your design pretty consistent and build a proper brand.

The cons of this approach is that you start off with tiniest screen and probably start with a really simple design which may not be very inspiring.

What are frameworks?

Frameworks are libraries with reusable code to design websites and applications. They give you a base to build on while still allowing you to be flexible with your layout and design.

Examples of front-end frameworks are:

  • Bootstrap
  • Foundation
  • Stylus
  • Semantic UI
  • UI Kit

Pros of using a framwork

  • Save time! With frameworks you can quickly and easily prototype designs
  • Utilise responsive design to allow your website to adapt to various screen sizes
  • Add consistency to design and code between projects and developers
  • Ensure cross-browser compatibility

Cons of using a framework

  • Using frameworks you will never understand the basics and won't get the satisfaction from creating something from scratch
  • Frameworks are often large and it is likely that you only need a fraction of what they come with
  • You never know how long a particular framework is supported, so you don't know how future-proof they are
